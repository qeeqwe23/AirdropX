# ADS-100 V8 平滑鲁棒方案

## 问题诊断

### V7的振荡问题

**现象**：
- 3秒周期振荡
- 高度在15-32m之间振荡
- 俯仰角在-30°到+25°之间剧烈摆动
- 控制指令在-0.4到+0.4之间大幅振荡（几乎饱和）

**根本原因**：
- **增益过载**：kp=1.6在90m/s高动压下产生过大的回路增益
- **高带宽**：omega_o=8.0导致控制器"反应过激"
- **舵面效能极高**：在高动压下，舵面效能极高，小的指令变化产生大的力矩
- **控制诱发振荡（CIO）**：控制器对高度偏差过于敏感，导致反复超调

**物理分析**：
- 控制器检测到高度偏差
- 输出大的控制指令
- 舵面快速响应，产生大的俯仰力矩
- 飞机快速纠偏，但产生超调
- 控制器检测到反向偏差
- 输出反向大的控制指令
- 循环往复，形成振荡

## V8 核心改进

### 1. 降低观测器带宽（Observer Bandwidth）

**V7的设置**：
```
omega_o = 8.0
beta_01 = 24.0
beta_02 = 192.0
beta_03 = 512.0
```

**V8的调整**：
```
omega_o = 4.0  (从8.0降至4.0)
beta_01 = 3 * omega_o = 12.0
beta_02 = 3 * (omega_o^2) = 48.0
beta_03 = omega_o^3 = 64.0
```

**效果**：
- 滤除气动弹性引起的虚假扰动
- 降低对测量噪声的敏感度
- 增加系统的相位裕度

### 2. 降低控制律增益（Controller Gains）

**V7的设置**：
```
kp = 1.6
kd = 3.5
```

**V8的调整**：
```
omega_c = 0.9
kp = omega_c^2 = 0.81  (从1.6降至0.81)
kd = 2 * omega_c = 1.80  (从3.5降至1.80)
```

**效果**：
- 降低系统对高度偏差的敏感度
- 放慢纠偏速度
- 让物理模型有时间响应舵面变化
- 建立稳定的阻尼

### 3. 提升数值阻尼（b0调整）

**V7的设置**：
```
b0 = 1.5
```

**V8的调整**：
```
b0 = 2.0  (从1.5提高至2.0)
```

**ADRC理论解释**：
- 增大b0意味着降低了算法对执行机构能力的预期
- 计算出的指令量会整体成比例缩小
- 作为一种全局安全系数

**效果**：
- 防止"打满舵"现象
- 增加数值阻尼
- 使控制更加保守和稳定

### 4. 引入变化率限制器（Rate Limiter）

**核心逻辑**：
```python
u_raw = (u0 - z3) / b0 + u_ff
u_diff = u_raw - u_prev
u_diff_clamped = clip(u_diff, -rate_limit, +rate_limit)
u_out = u_prev + u_diff_clamped
```

**参数设置**：
```
u_rate_limit = 0.02  (每帧最大变化量)
```

**物理意义**：
- 限制每帧舵角改变量不超过0.02
- 相当于约2.4°/s的变化率
- 升降舵从0到0.4至少持续20帧（0.16秒）

**效果**：
- 在软件层面模拟舵机的物理带宽限制
- 显著平滑俯仰角的突变
- 使高度变化更加柔和
- 防止控制指令的瞬时跳变

### 5. 调整前馈系数

**V7的设置**：
```
k_ff = 0.35
```

**V8的调整**：
```
k_ff = 0.28  (略微降低前馈强度)
```

**效果**：
- 降低投放瞬间的冲击
- 配合降低的控制增益

## V7 vs V8 对比

### V7高增益版（振荡）
```python
# 观测器
omega_o = 8.0
beta_01 = 24.0
beta_02 = 192.0
beta_03 = 512.0

# 控制律
kp = 1.6
kd = 3.5

```

**问题**：
- 反应过激
- 3秒周期振荡
- 俯仰角±30°摆动
- 控制指令饱和

### V8平滑鲁棒版（稳定）
```python
# 观测器
omega_o = 4.0
beta_01 = 12.0
beta_02 = 48.0
beta_03 = 64.0

# 控制律
kp = 0.81
kd = 1.80
b0 = 2.0

# 前馈
k_ff = 0.28

u_rate_limit = 0.02
```

**优势**：
- 反应平滑
- 消除振荡
- 俯仰角平稳
- 控制指令柔和

## 设计理念

### 通过"降频"换取"稳定性"

**核心思想**：
- 降低控制带宽
- 增加相位裕度
- 牺牲快速性，换取稳定性

**工程权衡**：
- V7：快速响应，但振荡
- V8：响应稍慢，但稳定

### 变化率限制的重要性

**物理本质**：
- 真实舵机有物理带宽限制
- 软件层面模拟这个限制
- 防止理想化的瞬时跳变

**工程实践**：
- 大多数飞控系统都有Rate Limiter
- 这是工程稳定性的关键

## 关键验证指标

| 检查项 | 目标值 (V8) | 验证逻辑 |
|--------|------------|---------|
| 高度波动幅值 | abs(H - 20.0) < 1.0 m | 消除V7的15-32m剧震 |
| 俯仰角摆动 | abs(Pitch) < 4.0 deg | 实现平稳平飞 |
| 舵面变化斜率 | du/dt < 2.4°/sec | 避免舵面瞬间跳变 |
| 投放瞬时超调 | Peak_H < 22.5 m | 1.2吨丢货后高度波动极小 |
| 稳定时间(Ts) | Ts < 1.2 s | 维持高效的ADRC收敛特性 |

## 实施要点

1. **降低观测器带宽**：omega_o从8.0降至4.0
2. **降低控制增益**：kp从1.6降至0.81，kd从3.5降至1.80
3. **增大b0**：从1.5增至2.0
4. **降低前馈系数**：k_ff从0.35降至0.28
5. **引入变化率限制**：u_rate_limit = 0.02
6. **保持V7的符号对齐**：e_h = h_meas - h_ref
7. **保持V7的强制状态归零**：z2 = 0.0

## 预期效果

在解决了极性错误（V7）后，现在的重点是**"降噪和平滑"**。一旦振荡消除，ADS-100将完全达到申报文档中的所有技术要求。
