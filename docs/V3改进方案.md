# ADS-100 V3 稳定版改进方案

## 问题诊断

当前系统存在的核心问题：

1. **高增益、无配平**：系统处于非稳态启动
2. **俯仰角失控**：±90度剧烈摆动
3. **舵面饱和**：长期处于+1.0饱和状态
4. **坠毁问题**：GEAR_CONTACT警告频繁出现

## V3 核心改进

### 1. 初始纵向配平（Longitudinal Trim）

**必要性**：
- HY100在150节平飞时需要非零配平舵角（如-0.05或+0.02）
- 不配平会导致第一帧就产生巨大力矩失衡
- 控制器为修正会迅速打满舵，进入非线性振荡

**实现方法**：
```
Step 1: 设置初始状态 -> H=100m, V=150kts
Step 2: 启动引擎 -> propulsion/set-running = 1
Step 3: 执行配平 -> simulation/do_trim = 1
Step 4: 获取配平舵位 -> elevator_trim
Step 5: 激活ADRC -> u_total = elevator_trim + u_adrc_delta
```

### 2. 大幅降低控制带宽

**原因**：大型无人机具有巨大旋转惯量，需要更保守的带宽

**新参数**：
```
omega_o = 6.0  (从12.0降至6.0，滤除高频噪声)
omega_c = 1.5  (从3.5降至1.5，防止指令过敏)

观测器增益：
beta_01 = 3 * omega_o = 18.0
beta_02 = 3 * (omega_o^2) = 108.0
beta_03 = omega_o^3 = 216.0

反馈增益：
kp = omega_c^2 = 2.25
kd = 2 * omega_c = 3.0

控制量系数：
b0_base = 1.0  (从0.65增至1.0，减弱敏感度)
```

### 3. 抑制舵饱和

**幅度限制**：ADRC增量输出限制在[-0.4, 0.4]

**原理**：
- 保证舵面永远不会死锁在±1.0
- 留出足够机械余量应对突发扰动

### 4. 带宽降低与稳定性

**omega_o = 6.0**：
- 观测器更"迟钝"
- 对JSBSim数值噪声有更好免疫力

**omega_c = 1.5**：
- 减慢控制响应
- 几吨重飞机姿态改变物理上需要1-2秒
- 之前3.5带宽超出物理响应极限，导致相位滞后振荡

### 5. 软启动与指令斜坡

**策略**：
- 先让飞机在100m稳定飞行5秒
- 以2.0 m/s速度缓慢降低目标高度

**公式**：
```
h_target(t) = max(20.0, 100.0 - 2.0 * t)
```

## V3 验证指标

| 指标项 | 目标值 | 状态预期 |
|--------|--------|---------|
| 初始高度波动 | abs(H_error) < 0.5 m | 稳定通过 |
| 俯仰角波动 | abs(Pitch) < 3.0 deg | 不再剧震 |
| 舵面状态 | abs(Elev_cmd) < 0.5 | 远离饱和 |
| 离地警告 | GEAR_CONTACT = FALSE | 消除坠毁 |

## 实施计划

1. 创建ADRC控制器V3（低带宽版本）
2. 修改JSBSimWrapper添加初始配平逻辑
3. 实现软启动和指令斜坡功能
4. 运行测试验证100m平飞稳定性
5. 验证通过后再测试空投功能

## 设计原则

遵循航空控制系统开发的**"先稳态、后动态"**原则：
1. 先在100米高度验证平飞
2. 待高度和姿态曲线水平
3. 再触发1.2吨空投指令
