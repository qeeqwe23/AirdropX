## 系统概述

ADS-100 是一个专业的无人机空投控制系统，用于 HY100 大型无人机的精准空投任务。系统采用自抗扰控制（ADRC）算法，能够在载荷突变情况下保持高度稳定。

### 主要功能

- **实时仿真**: 基于 JSBSim 的高保真飞行动力学仿真
- **智能控制**: ADRC 算法自动补偿载荷投放扰动
- **数据可视化**: 实时显示飞行参数和控制指令
- **性能分析**: 自动评估系统性能指标

## 快速开始

### 启动程序

1. 打开终端，进入项目目录
2. 激活虚拟环境：
   ```bash
   source ads100_env/bin/activate
   ```
3. 运行主程序：
   ```bash
   python main.py
   ```

### 首次使用

1. 系统启动后，默认显示待机状态
2. 左侧面板显示任务配置参数
3. 中央面板显示实时监控曲线
4. 右侧面板显示精度分析结果

## 界面说明

### 顶部标题栏

- **系统标题**: ADS-100 海雀空投仿真与测控平台 v1.0
- **任务状态**: 显示当前系统状态（待机/运行中/已停止）
- **时间显示**: 当前系统时间
- **仿真频率**: 显示仿真运行频率（目标 120Hz）

### 左侧配置面板

#### 载荷管理
- **总载重**: 设置货物总质量（范围 0-5000kg）
  - 默认值：1200kg（4 件 × 300kg）

#### 投放模式
- **模式选择**: 
  - 单件投放：每次投放一件货物
  - 4 件连投：按设定间隔连续投放 4 件货物
- **间隔时间**: 连投模式下的投放间隔（范围 0.1-10.0s）
  - 默认值：0.50s

#### 环境建模
- **高度**: 设置飞行高度（范围 10-500m）
  - 默认值：100m
  - 注意：超低空（< 50m）时地面效应明显
- **风速**: 设置环境风速（范围 0-30 m/s）
  - 默认值：8.5 m/s
- **风向**: 设置风向角度（范围 0-360°）
  - 默认值：270°（西风）

#### 控制按钮
- **启动系统仿真**: 初始化并启动仿真
- **执行物资投放 (SIGNAL)**: 手动触发投放
- **停止仿真**: 停止当前仿真
- **重置系统**: 清空数据并重置到初始状态

### 中央监控面板

#### 高度监控标签页
- **实际高度曲线**（绿色）: 显示飞机实时高度
- **目标高度线**（蓝色虚线）: 显示目标高度
- **上下限线**（红色点线）: ±4m 性能指标边界
- **投放时刻标记**（橙色虚线）: 标记投放发生时刻

#### 俯仰角监控标签页
- 显示飞机俯仰角变化
- 用于评估飞行姿态稳定性

#### 控制指令标签页
- 显示升降舵控制指令（归一化值 -1 到 +1）
- 正值表示抬头，负值表示低头

#### 质量重心标签页
- **质量曲线**（紫色）: 显示全机质量变化
- **重心曲线**（黄色）: 显示重心纵向位置
- 投放瞬间可见明显跳变

### 右侧分析面板

#### 蒙特卡洛落点散布
- 显示多次投放的落点分布
- CEP 圆（蓝色虚线）: 50% 概率圆偏差

#### 精度指标
- **CEP 50%**: 圆概率误差（单位：米）
- **最大高度偏差**: 投放后的最大高度偏差
- **稳定时间**: 恢复到误差带的时间
- **结论**: 自动判定合格/不合格

#### 任务事件日志
- 记录系统关键事件
- 显示时间戳和事件描述

### 底部状态栏

显示实时飞行参数：
- 经度、纬度
- 空速、地速
- 高度
- 重心位置

## 操作流程

### 标准操作流程

#### 1. 配置任务参数

```
步骤 1: 设置载荷参数
- 总载重：1200 kg
- 投放模式：4 件连投
- 间隔时间：0.50 s

步骤 2: 设置环境参数
- 高度：100 m
- 风速：8.5 m/s
- 风向：270 deg

步骤 3: 确认配置
- 检查所有参数是否符合任务要求
```

#### 2. 启动仿真

```
步骤 1: 点击"启动系统仿真"按钮
步骤 2: 等待系统初始化
- 观察事件日志显示"系统初始化完成"
- 任务状态变为"运行中"（绿色）
步骤 3: 观察飞行数据
- 监控面板开始显示实时曲线
- 状态栏显示飞行参数
```

#### 3. 执行投放

```
步骤 1: 等待飞行稳定
- 观察高度曲线接近目标高度
- 等待 3-5 秒让系统稳定

步骤 2: 触发投放
- 点击"执行物资投放 (SIGNAL)"按钮
- 观察事件日志显示投放信息

步骤 3: 观察系统响应
- 监控高度曲线变化
- 观察控制指令调整
- 查看质量和重心跳变
```

#### 4. 分析结果

```
步骤 1: 等待仿真完成
- 让系统运行 10-15 秒
- 观察高度是否恢复稳定

步骤 2: 查看性能指标
- 检查最大高度偏差
- 确认稳定时间
- 查看结论判定

步骤 3: 导出数据（可选）
- 截图保存曲线
- 记录性能指标
```

#### 5. 停止和重置

```
步骤 1: 停止仿真
- 点击"停止仿真"按钮
- 任务状态变为"已停止"（红色）

步骤 2: 重置系统
- 点击"重置系统"按钮
- 清空所有曲线和数据
- 准备下一次仿真
```

### 高级操作

#### 多次投放测试

如需测试多次投放：

1. 启动仿真
2. 等待 5 秒
3. 点击投放按钮（第 1 次）
4. 等待 10 秒观察恢复
5. 点击投放按钮（第 2 次）
6. 重复直到 4 件货物全部投放

#### 参数优化

调整控制参数以优化性能：

1. 修改 `core/adrc_controller.py` 中的增益参数
2. 重新运行测试脚本验证性能
3. 观察高度保持精度和稳定时间
4. 迭代优化直到满足指标

## 性能指标

### PR-H-01: 超低空高度保持精度

**定义**: 无投放情况下的高度跟踪误差

**要求**: ≤ ±2.0m

**验证方法**:
1. 启动仿真但不投放
2. 运行 30 秒
3. 记录最大高度偏差
4. 判定是否满足要求

**合格标准**: 
- ✓ 合格：最大偏差 ≤ 2.0m
- ✗ 不合格：最大偏差 > 2.0m

### PR-H-02: 连投瞬间高度波动

**定义**: 投放后的最大高度偏差

**要求**: ≤ ±4.0m

**验证方法**:
1. 执行投放操作
2. 记录投放后 10 秒内的最大偏差
3. 判定是否满足要求

**合格标准**:
- ✓ 合格：投放后最大偏差 ≤ 4.0m
- ✗ 不合格：投放后最大偏差 > 4.0m

### PR-F-01: 仿真主频

**定义**: 仿真运行的实际频率

**要求**: ≥ 120Hz

**验证方法**:
1. 运行仿真
2. 计算：实际频率 = 仿真步数 / 仿真时间
3. 判定是否满足要求

**合格标准**:
- ✓ 合格：实际频率 ≥ 120Hz
- ✗ 不合格：实际频率 < 120Hz

### 稳定时间

**定义**: 投放后恢复到 ±2m 误差带的时间

**要求**: ≤ 2.0s

**验证方法**:
1. 执行投放操作
2. 记录高度首次进入 ±2m 误差带的时间
3. 判定是否满足要求

**合格标准**:
- ✓ 合格：稳定时间 ≤ 2.0s
- ✗ 不合格：稳定时间 > 2.0s

## 故障排除

### 常见问题

#### 1. 程序无法启动

**症状**: 双击运行无响应或报错

**解决方案**:
```bash
# 检查 Python 版本
python --version  # 应该 >= 3.8

# 检查虚拟环境
source ads100_env/bin/activate

# 检查依赖
pip list | grep PyQt6
pip list | grep jsbsim

# 重新安装依赖
pip install -r requirements.txt
```

#### 2. JSBSim 初始化失败

**症状**: 事件日志显示"JSBSim 初始化失败"

**解决方案**:
```bash
# 检查飞机模型文件
ls aircraft/HY100/HY100.xml
ls aircraft/HY100/reset.xml

# 检查 jsbsim 安装
python -c "import jsbsim; print(jsbsim.__version__)"

# 重新安装 jsbsim
pip uninstall jsbsim
pip install jsbsim==1.1.13
```

#### 3. 界面显示异常

**症状**: 曲线不更新或界面卡顿

**解决方案**:
- 检查仿真是否正在运行
- 点击"停止仿真"后重新启动
- 点击"重置系统"清空数据
- 重启程序

#### 4. 性能指标不合格

**症状**: 高度偏差过大或稳定时间过长

**原因分析**:
- 控制参数未针对当前配置优化
- 初始条件设置不合理
- 环境参数（风速、高度）超出设计范围

**解决方案**:
- 调整 ADRC 控制器增益参数
- 修改初始高度（建议 ≥ 100m）
- 降低风速（建议 ≤ 10 m/s）
- 参考设计文档重新整定参数

#### 5. 仿真频率过低

**症状**: PR-F-01 不合格，频率 < 120Hz

**原因分析**:
- 计算机性能不足
- 后台程序占用资源

**解决方案**:
- 关闭其他占用 CPU 的程序
- 降低仿真时长
- 使用性能更好的计算机
