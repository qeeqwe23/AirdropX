# AirdropX Task-C 阶段二：有风 CARP/CEP 纵向偏差修正与侧风分析报告

## 1. 验证目标

在 Task-C 阶段一中，我们发现 `carp20_v1` 求解器在有风条件下（如 8m/s 顺逆风）存在显著的纵向系统偏差（bias_N 达到 14.24m）。
本阶段（阶段二）的主要目标：
1. **TC-C2 诊断根因**：分析并定位纵向系统偏差的物理和算法根源。
2. **TC-C3 算法修正**：修正 CARP 风补偿公式（升级至 `carp20_v2`），重新进行 16 组蒙特卡洛扫描，验证 bias_N 的收敛情况。
3. **TC-C4 侧风 CARP 静态偏移补偿验证**：分析侧风（90°/270°）场景下的横向偏差（bias_E），验证静态释放点偏移补偿的有效性。
4. **TC-C4 UI 接入**：在 UI 界面中接入 `carp20_v2` 求解器，并实时显示 Release_N、CEP50 以及 PASS/FAIL 结论。

## 2. 验证结果

### 2.1 TC-C2 纵向偏差根因诊断

通过分析 `carp20_v1` 的风补偿公式，发现**阻力修正项**存在物理模型错误：
- **错误逻辑**：v1 版本基于**地速**计算货物弹道前飞距离，导致在顺风/逆风时，阻力衰减被错误地放大或缩小。
- **物理事实**：货物在空中飞行时，继承的是载机的**空速**（相对于空气的速度），空气阻力也只与空速相关。风偏移应作为独立的叠加项（风速 × 下落时间）。
- **验证**：在 8m/s 逆风下，地速为 70.6m/s，v1 算出的前飞距离比实际少了约 14.56m，这与实测的 bias_N = +14.24m 高度吻合。

### 2.2 TC-C2 算法修正与验证：★ ALL PASS ★

基于上述诊断，开发了 `carp20_v2` 求解器，实现了**空速弹道与风偏移严格分离**：
1. 弹道前飞距离基于**空速**计算。
2. 阻力修正系数基于**空速**计算。
3. 风偏移（wind_drift）独立累加。

**验证结果（完整 16 组 0/3/5/8m/s × 0/90/180/270° 风场扫描，每组 320 样本）：**
- **最大 CEP50**：由 14.278m 降至 **1.080m**（改善 13.2x）。
- **最大 |bias_N|**：由 14.240m 降至 **0.626m**（改善 22.7x）。
- **结论**：TC-C2 纵向风补偿 PASS。carp20_v2 已解决顺逆风条件下由地速/空速混用造成的系统性纵向偏差。所有 16 组场景的 CEP50 均远低于 20m 门限。

### 2.3 TC-C3 侧风 CARP 静态偏移补偿验证：★ ALL PASS ★

在 0m/s、5m/s 和 8m/s 的纯侧风（90°东风、270°西风）5 组专项场景下进行分析。
由于 MQ-9 JSBSim 模型无横向 FCS，直接注入副翼指令会导致持续滚转失稳。因此，本轮侧风横向精度通过 `carp20_v2` 的静态释放点偏移补偿实现（基于实测系数 $k\_side=0.0776$），**而非真正的飞机横向闭环航迹控制器**。

**实测结果（320样本）：**
- **最大横向偏差（|bias_E|）**：仅为 **0.0184m**。
- **最大 Cross-track P95**：**0.0858m**（远低于 5.0m 门限）。
- **最大 CEP50**：**3.213m**（远低于 20.0m 门限）。
- **结论**：在当前 20m 无伞投放、固定航向和 8m/s 以下风速条件下，静态释放点侧偏补偿已能满足落点 CEP 要求；**飞机横向闭环航迹控制仍作为后续高动态任务和更复杂航线任务的研究项保留。**

### 2.4 TC-C4 UI 接入与 Gate 验证：★ ALL PASS ★

在 UI `simulation_thread.py` 和 `analysis_panel.py` 中成功接入了 `carp20_v2` 求解器。

**Gate 验证结果：**
- **G1 (UI 显示 Release_N)**：PASS。界面中新增 `release_n_label`，实时显示 v2 求解器计算的释放点坐标。
- **G2 (Release_N 计算正确性)**：PASS。无风 49.24m，8m/s 北风 65.39m，误差 < 0.01m。
- **G3 (PASS/FAIL 标签显示)**：PASS。根据 CEP50_to_target 自动判定，并在界面显示大字 PASS/FAIL。
- **G4 (NW20 模式保护)**：PASS。NW20 高度稳定基线模式自动跳过 CEP 评价，不破坏已有基线。

## 3. 总体结论

**AirdropX v2.5 完成 Task-C 阶段二验证。**

在 v2.3 NW20 高度稳定基线与 v2.4 Task-B 无风 CARP/CEP 基线不退化的前提下，系统完成了 carp20_v2 风补偿求解器升级。通过空速弹道与风场漂移分离，顺逆风纵向系统偏差由阶段一最差约 14.24m 收敛至最大约 0.626m；16 组 0/3/5/8m/s × 0/90/180/270° 风场扫描全部通过，最大 CEP50 为 1.080m，满足 20m 无伞投放精度门限。UI 已接入 v2 求解器，能够显示 Release_N 与 CEP PASS/FAIL，并保持 NW20 模式不受影响。

**验证边界声明：**
本结论覆盖 8m/s 以下静态风场、20m 无伞、固定目标点、MQ-9 代理模型和 JSBSim 仿真条件下的 CARP/CEP 精度验证；**不覆盖 HY100 实机能力、阵风/湍流环境、有伞投放和复杂航迹闭环控制。**

建议将当前版本封版为：**AirdropX v2.5：8m/s 以下静态风场 CARP/CEP 后端与 UI 接入验证版**，并进入下一阶段 Task-D：鲁棒性与工程化验收。
