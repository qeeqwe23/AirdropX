# ADS-100 V7 符号对齐方案

## 核心问题诊断

### 致命错误：符号定义冲突

**JSBSim标准定义**：
- `fcs/elevator-cmd-norm` 升降舵指令
- **正值（+）**：下压头（Pitch Down）
- **负值（-）**：上抬头（Pitch Up）

**V6代码的错误逻辑**：
```python
u0 = kp * (h_ref - z1)  # 目标高度 - 实际高度
```

**问题链条**：
1. 当飞机高于目标高度：`z1 > h_ref`
2. 误差为负：`u0 < 0`
3. 输出负值升降舵指令
4. **结果**：上抬头！
5. **后果**：飞机越飞越高

**这就是为什么飞机从20m爬升到160m的根本原因！**

### 状态注入的负面反馈

**V6的问题**：
1. 预运行10帧，配平未成功
2. 飞机产生2.04 m/s的爬升速度
3. 状态注入捕获这个速度：`z2 = 2.04`
4. 由于符号错误，控制器认为这是"需要维持"的爬升
5. 锁定了错误趋势

## V7 核心改进

### 1. 符号对齐控制律（Sign-Aligned Control）

**正确的误差定义**：
```
e_h = h_meas - h_ref    (实测高度 - 目标高度)
v_up = v_z_up           (向上速度为正)
```

**正确的控制指令**：
```
u_adrc = kp * e_h + kd * v_up - z3
```

**逻辑校验**：
- 如果飞机太高（e_h > 0）：u_adrc为正 → 下压头（Pitch Down）→ ✓ 正确
- 如果飞机爬升（v_up > 0）：u_adrc为正 → 下压头（阻尼效果）→ ✓ 正确

### 2. 强制初始状态归零（Hard Reset）

**取消预运行**：
```python
# 不再执行预运行10帧
# 直接在静止状态执行配平
```

**强制初始条件**：
```python
self.fdm['ic/v-down-fps'] = 0.0  # 强制垂直速度为0
self.fdm['ic/q-degps'] = 0.0     # 强制俯仰速率为0
```

**强制状态注入**：
```python
def reset_states(self, h_initial):
    self.z1 = h_initial
    self.z2 = 0.0  # 强制初始观测速度为0，不接受爬升惯性
    self.z3 = 0.0
```

**效果**：
- 即使JSBSim第一帧有微小抖动
- 控制器认为初始状态必须是平飞
- 立即通过`kd * v_up`抑制任何爬升倾向

### 3. 投放前馈方向修正

**物理分析**：
- 1.2吨物资离机
- 飞机变轻，升力相对增大
- 飞机会向上蹿

**V6的错误**：
```python
u_ff = -k_ff * (delta_m / m_total)  # 负值，上抬头
```

**V7的修正**：
```python
u_ff = k_ff * (delta_m / m_total)  # 正值，下压头
```

**效果**：
- 提前下压舵抵消升力增量
- 将稳定时间锁定在2s以内

### 4. 调整配平备用值

**V6的问题**：
```python
self.trim_elevator = 0.02  # 可能不足
```

**V7的改进**：
```python
self.trim_elevator = 0.035  # 增大到0.035，针对20m地效区
```

### 5. 调整b0参数

**V6的设置**：
```python
self.b0 = 1.8
```

**V7的调整**：
```python
self.b0 = 1.5  # 对应配平舵位响应灵敏度
```

### 6. 调整前馈系数

**V6的设置**：
```python
self.k_ff = 0.30
```

**V7的调整**：
```python
self.k_ff = 0.35  # 增大到0.35
```

## V6 vs V7 对比

### V6错误的控制律
```python
# 误差定义
e1 = h_ref - self.z1  # 目标 - 实际

# 控制指令
u0 = self.kp * e1 + self.kd * (0.0 - v_z)

# 前馈
u_ff = -self.k_ff * (delta_m / m_total)  # 负值
```

**问题**：
- 飞机太高时输出负值（上抬头）
- 投放后输出负值（上抬头）
- 完全反向！

### V7正确的控制律
```python
# 误差定义
e_h = h_meas - h_ref  # 实际 - 目标

# 控制指令
u0 = self.kp * e_h + self.kd * v_up

# 前馈
u_ff = k_ff * (m_drop / m_total)  # 正值
```

**优势**：
- 飞机太高时输出正值（下压头）
- 投放后输出正值（下压头）
- 符号对齐！

## 关键指标校验表

| 检查项 | 理想状态 | 逻辑说明 |
|--------|---------|---------|
| H > 20m 时指令 | u_adrc 应为正值 (+) | 正值代表下压舵(Pitch Down) |
| V_up > 0 时指令 | u_adrc 应为正值 (+) | 阻尼项应下压机头 |
| 初始 z2 值 | z2 == 0.0 | 强制消除初始爬升惯性 |
| 1.2t 投放前馈 | u_ff 为正值 (+) | 提前下压舵抵消升力 |

## 实施要点

1. **符号对齐**：误差定义改为 `e_h = h_meas - h_ref`
2. **取消预运行**：不再执行预运行帧
3. **强制初始状态**：`ic/v-down-fps = 0.0`, `ic/q-degps = 0.0`
4. **强制状态注入**：`z2 = 0.0`（不接受爬升惯性）
5. **前馈方向修正**：投放前馈改为正值
6. **调整配平备用值**：0.035（针对20m地效区）
7. **调整b0**：1.5
8. **调整k_ff**：0.35

## 预期效果

纠正了升降舵正负极性后，系统将从"发散爬升"转变为"向心回归"，彻底解决20m超低空任务中的稳定性问题。
