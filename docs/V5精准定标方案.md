# ADS-100 V5 精准定标方案

## 问题诊断

V4版本已解决坠毁和控制饱和问题，当前重点：

1. **消除初始化瞬态冲击**：初始高度104.95m vs 目标100m产生大误差
2. **提升丢载后响应速度**：稳定时间3.29s，需要<2.0s
3. **切换到20m超低空任务**：符合项目要求

## V5 核心改进

### 1. 软启动逻辑（Soft-Start）

**问题**：
- V4在第0秒时，实际高度104.95m而目标100m
- 控制器产生巨大下压指令导致剧烈震荡

**解决方案**：
```python
if self.sim_time < 2.0:
    current_target = h_meas  # 前2秒锁定为当前高度
else:
    current_target = self.target_h  # 2秒后切换到任务高度
```

**效果**：
- 前2秒以当前实际高度为基准平稳对接
- ESO观测器状态稳定后再切回任务高度
- 彻底消除初始"跳变"

### 2. 20m超低空任务对标

**物理特性**：
- 20m飞行时，JSBSim地面效应模型自动增加升力线斜率

**算法应对**：
- ADRC的z3项自动捕获地效产生的"额外升力"
- 将其作为扰动对冲
- 直接验证项目核心指标

**初始条件设置**：
```
ic/h-agl-ft = 65.6  (20m)
ic/ve-kts = 175.0   (约90 m/s)
```

### 3. 强化反馈控制律

**比例增益提升**：
```
kp = 1.6  (从1.0提升，加快误差收敛)
kd = 2.8  (从2.0提升，提供强阻尼，抑制超调)
```

**控制律公式**：
```
u0 = kp * (h_ref - z1) + kd * (0 - v_z)
```

**效果**：
- 缩短丢载后恢复时间
- 强微分项更早感知高度上升趋势
- 提前抑制，将稳定时间缩短至<2.0s

### 4. 观测器带宽优化

**提升观测器带宽**：
```
omega_o = 7.0  (从6.0提升)

观测器增益：
beta_01 = 3 * omega_o = 21.0
beta_02 = 3 * (omega_o^2) = 147.0
beta_03 = omega_o^3 = 343.0
```

### 5. 前馈参数优化

**优化前馈系数**：
```
k_ff = 0.38  (从0.35微调)
u_ff = - k_ff * (delta_m / m_total)
```

### 6. 控制量系数调整

**降低b0**：
```
b0 = 1.5  (从2.0降至1.5)
```

**原因**：提高控制灵敏度，配合增大的kp和kd

### 7. 输出限幅调整

**调整限幅范围**：
```
u_prev = np.clip(u_total, -0.5, 0.5)
```

### 8. 强制配平序列优化

**简化预运行**：
```
for _ in range(5): self.fdm.run()  # 从120帧减至5帧
```

**原因**：减少预运行期间的高度变化

## V5 关键验证指标

| 指标项 | 目标值 | 验证方法 |
|--------|--------|---------|
| 初始对接稳定性 | abs(H - 20.0) < 0.1 m | 观察0-2s软启动曲线 |
| 投放瞬时超调 | Peak_H < 22.5 m | 1.2t丢货后高度波动 |
| 稳定时间(Ts) | Ts < 1.8 s | 误差进入±0.5m区域时间 |
| 俯仰角范围 | abs(Pitch) < 4.0 deg | 丢货瞬间姿态变化 |
| 配平有效性 | trim_elev != 0.0 | 检查初始化日志输出 |

## 改进总结

V5是最接近真机实飞逻辑的版本：

1. **软启动**解决能量状态不对等产生的初始误差
2. **20m超低空**符合项目申报书要求
3. **强化增益**提升响应速度
4. **优化观测器**提高扰动观测能力
5. **精准配平**确保稳定启动
