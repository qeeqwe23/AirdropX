# ADS-100 V6 零漂移方案

## 问题诊断

V5版本的软启动逻辑变成了"跟随逻辑（Follow-Me）"，导致系统处于开环漂移状态：

**根本问题**：
```python
if soft_start_active:
    current_target = h_current  # 目标等于当前高度
```

**导致的后果**：
- 误差 e1 = target_h - h_meas = 0（永远为0）
- 控制器完全放弃了对高度的约束
- 飞机仅凭初始配平舵位（-0.05）在175节高动压下产生巨大爬升力矩
- 20m处存在强烈地面效应，飞机像"气球"一样飘升

## V6 核心改进

### 1. 弃用软启动跟随逻辑

**V5的错误**：
- 动态改变目标高度：target = h_meas
- 误差永远为0，控制器失去约束力

**V6的正确做法**：
- 从第0秒起锁定 target_h = 20.0
- 通过状态注入消除初始误差

### 2. 控制器状态注入（State Injection）

**在 t=0 时**：
```python
z1 = h_meas(0)    # 将观测高度直接对齐实际高度
z2 = v_z(0)       # 将观测垂直速度对齐实际速度
z3 = 0.0          # 初始化扰动估计为零
```

**效果**：
- 初始误差为0，但控制器的纠偏能力100%激活
- 一旦飞机偏离0.01m，控制器立刻反应
- 不像V5那样视而不见

### 3. 精准自动配平（Auto-Trim）

**V5的问题**：
- 手动硬编码 trim_elevator = -0.05
- 该值可能在100m适用，但在20m地效区绝对错误

**V6的改进**：
```python
# 预运行10帧让物理引擎计算动压
for _ in range(10): self.fdm.run()

# 强制执行纵向配平
self.fdm['simulation/do_trim'] = 1

# 捕获JSBSim自动计算的精准配平舵位
self.trim_elevator = self.fdm['fcs/elevator-pos-norm']

# 验证配平是否成功
if abs(self.trim_elevator) < 1e-5:
    # 备用：根据动压给定经验俯冲角抵消地效
    self.trim_elevator = 0.02  # HY100在地效区需要微量下压舵
```

**优势**：
- JSBSim自动计算抵消地效所需的精准舵位
- 通常是一个微小的下压舵角
- 不再依赖手动猜测

### 4. 阻尼增强（Enhanced Damping）

**提升观测器带宽**：
```
omega_o = 8.0  (从7.0提升)

观测器增益：
beta_01 = 3 * omega_o = 24.0
beta_02 = 3 * (omega_o^2) = 192.0
beta_03 = omega_o^3 = 512.0
```

**强化微分增益**：
```
kp = 1.6  (保持不变)
kd = 3.5  (从2.8提升，增强阻尼)
```

**效果**：
- 相当于在飞机上挂了一个"阻尼器"
- 只要飞机有爬升趋势，控制器立刻产生反向力矩

### 5. 调整控制量系数

**增大b0**：
```
b0 = 1.8  (从1.5增至1.8)
```

**原因**：
- 降低单次控制量
- 使运动更平滑

### 6. 降低前馈系数

**调整前馈**：
```
k_ff = 0.30  (从0.38降至0.30)
```

**原因**：
- 防止因前馈过大导致下砸

## V6 关键验证指标

| 指标项 | 目标值 | 验证方法 |
|--------|--------|---------|
| 初始高度漂移 | abs(H - 20.0) < 0.2 m | 观察0-10s高度曲线 |
| 0-5s爬升率 | v_z < 0.1 m/s | 验证配平与闭环锁定能力 |
| 投放瞬时超调 | Peak_H < 22.0 m | 1.2t丢货后的最高高度 |
| 稳定时间(Ts) | Ts < 1.7 s | 误差进入±0.5m区域 |
| 控制器状态注入 | z1(0) == h_meas(0) | 检查初始化日志 |

## V5 vs V6 对比

### V5软启动（错误）
```python
if soft_start_active:
    current_target = h_current  # 目标跟随当前值
else:
    current_target = target_altitude
```

**问题**：
- 误差永远为0
- 控制器失去约束力
- 系统开环漂移

### V6状态注入（正确）

```python
# 初始化时注入状态
if not self.sim_started:
    self.controller.init_states(h_meas, v_z)
    self.sim_started = True

# 始终锁定目标高度
u_delta = self.controller.update(self.target_h, h_meas, v_z, m_dropped, m_total)
```

**优势**：
- 目标始终锁定20.0m
- 初始误差为0（通过状态注入）
- 控制器100%激活，随时准备纠偏

## 实施要点

1. 移除软启动逻辑
2. 实现控制器状态注入函数 init_states()
3. 增加预运行帧数到10帧
4. 使用JSBSim自动配平，备用值改为0.02
5. 提升观测器带宽到8.0
6. 强化微分增益到3.5
7. 增大b0到1.8
8. 降低前馈系数到0.30
9. 从第0秒起锁定目标高度为20.0m

## 预期效果

通过"控制器状态注入"代替"目标跟随"，确保无人机在20m处像贴在地面上一样平稳，为后续1.2吨空投任务打下完美的初始状态。
